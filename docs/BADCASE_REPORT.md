# Badcase 分析报告

## 1. Badcase 目标

本报告记录 WebKnow 测试过程中发现的问题，并从产品角度分析原因与优化方案。

---

## Badcase 1：操作类问题误拒答

### 用户问题

```text
VPN 和工具怎么用？
```

### 现象

系统最终回复：

```text
该网站暂未收录该信息
```

### 预期

知识库中存在 VPN、工具、Shadowrocket、TikTok、QQ 等相关操作内容，系统应基于知识库给出操作说明。

### 原因分析

判断知识库是否足够的 LLM 节点过于严格。它可能认为知识库中没有完整标题或定义，因此输出 NO。

### 优化方案

在判断 Prompt 中增加操作类规则：

```text
如果用户问的是“怎么做 / 怎么用 / 在哪里 / 如何操作”，只要检索结果中包含相关操作步骤、工具名称、页面路径、注意事项或案例，就输出 YES。
```

## Badcase 2：泛相关内容误判为可回答

### 用户问题

```text
AI 编造事实怎么办？
```

### 现象

系统走知识库回答分支，但最终回答说知识库没有明确答案。

### 预期

知识库检索结果只是泛 AI 内容，与“编造事实/幻觉”无关，应判断为 NO，并继续走 sitemap 实时补充。

### 原因分析

判断节点看到“AI”关键词后，把泛相关内容误判为可以回答。

### 优化方案

加入更严格的相关性判断规则：

```text
如果检索结果只是和用户问题属于同一大类，但没有回答用户真正关心的问题，输出 NO。
如果检索结果只是出现相同关键词，例如只出现“AI”，但没有涉及用户问题中的核心概念，输出 NO。
如果用户问题涉及“编造事实、幻觉、事实错误、无依据回答、hallucination”，但检索结果没有涉及这些概念或解决方法，输出 NO。
```

## Badcase 3：平台规则合规风险

### 用户问题

```text
TikTok 想看直播提示：您已违反规定。应该怎么办？
```

### 现象

系统建议重新注册账号、使用新设备和 VPN。

### 风险

该回答可能被理解为规避平台规则或绕过风控。

### 优化方案

在回答 Prompt 中加入合规边界：

```text
如果用户问题涉及平台违规、账号封禁、直播限制、风控、审核失败等情况，只能提供合规处理建议。
可以建议用户查看违规原因、停止违规行为、检查账号状态、提交申诉、联系平台客服。
不要建议用户通过换设备、换账号、换邮箱、使用 VPN、绕过风控、规避平台规则等方式解决问题。
```

---

## Badcase 4：来源 URL 过长或带多余符号

### 现象

回答中的参考来源有时出现：

```text
{https://murraytsai.github.io/...}
```

或中文 URL 编码过长，影响阅读体验。

### 原因分析

LLM 输出或代码节点未完全清洗 URL。

### 优化方案

在代码执行节点中统一清洗 URL：

- 去除 `{}`。
- 去除引号。
- 去除 Markdown 代码块。
- 只保留第一个有效 URL。

回答 Prompt 中加入：

```text
参考来源只输出纯 URL，不要带大括号、引号、代码块或多余符号。
```

---

## 5. Badcase 总结

| 问题类型 | 影响 | 优化方向 |
|---|---|---|
| 判断过严 | 有答案被拒答 | 优化操作类判断规则 |
| 判断过宽 | 泛相关内容被误判 | 强化核心概念匹配 |
| 合规风险 | 可能输出规避规则建议 | 增加安全边界 Prompt |
| 来源格式问题 | 影响可读性 | URL 清洗 |

这些 badcase 体现了 AI 产品上线前必须进行评估和迭代，而不是只关注成功案例。
