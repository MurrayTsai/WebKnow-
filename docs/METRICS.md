# 评估指标设计

## 1. 为什么需要指标

AI 产品经理不能只说“效果不错”，需要定义可评估指标，用于判断产品是否真实可用。

---

## 2. 核心指标

| 指标 | 定义 | 计算方式 |
|---|---|---|
| 知识库命中率 | 问题由知识库直接回答的比例 | 知识库回答数 / 总问题数 |
| 实时补充成功率 | 知识库不足时，sitemap + Jina 成功回答的比例 | 实时补充成功数 / 知识库不足问题数 |
| 无答案拒答准确率 | 无站内内容时正确拒答的比例 | 正确拒答数 / 无答案问题数 |
| 幻觉率 | 出现无依据回答的比例 | 幻觉回答数 / 总问题数 |
| 来源引用覆盖率 | 回答带来源的比例 | 带来源回答数 / 总回答数 |
| badcase 修复率 | 已发现问题中完成修复的比例 | 已修复 badcase 数 / 总 badcase 数 |

---

## 3. 当前测试集结果

基于 10 条测试用例：

| 结果类型 | 数量 |
|---|---:|
| 知识库直接回答 | 3 |
| sitemap + Jina 实时补充 | 4 |
| 正确拒答 | 1 |
| 需优化 badcase | 2 |

初步结论：

- 主流程已跑通。
- 知识库回答、实时补充、无答案拒答均已验证。
- 当前主要优化方向集中在相关性判断和合规边界。

---

## 4. 后续指标计划

后续可以扩展到 30-50 条测试集，并按以下类型覆盖：

- 精确事实问答
- 概念解释
- 跨文章总结
- 对比型问题
- 操作流程问题
- 无答案问题
- 幻觉诱导问题
- 合规风险问题
