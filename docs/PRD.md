# PRD：WebKnow 准实时网站知识库问答助手

## 1. 产品概述

### 1.1 产品名称

WebKnow：准实时网站知识库问答助手

### 1.2 产品定位

WebKnow 是一个面向博客、文档站、企业官网和知识型网站的 AI 问答助手。它通过 Dify Knowledge 构建网站知识库，并结合 sitemap 与 Jina Reader 实现站内内容的准实时补充回答。

### 1.3 一句话描述

当用户询问网站内容时，WebKnow 优先从知识库回答；如果知识库没有答案，则读取 sitemap 并实时抓取相关文章内容进行补充；如果站内也没有答案，则明确拒答。

---

## 2. 背景与问题

### 2.1 背景

许多网站内容丰富，但用户很难快速找到答案。传统搜索依赖关键词匹配，不能理解用户问题。普通 RAG 问答虽然可以提升回答质量，但知识库必须提前导入，一旦网站新增内容或导入不完整，就可能无法回答。

### 2.2 用户痛点

| 用户痛点 | 说明 |
|---|---|
| 找内容慢 | 用户需要手动翻文章、标签、归档页 |
| 搜索体验差 | 传统站内搜索只匹配关键词 |
| 普通 RAG 更新慢 | 新文章未进入知识库时无法回答 |
| AI 容易幻觉 | 没有资料时可能编造答案 |
| 无法追溯来源 | 用户不知道答案来自哪篇文章 |

## 4. 产品目标

### 4.1 V1.5 目标

- 实现基于知识库的网站问答。
- 当知识库不足时，通过 sitemap 找到站内相关文章。
- 使用 Jina Reader 实时读取文章正文。
- 基于站内内容回答，避免无依据编造。
- 站内没有内容时明确拒答。

### 4.2 非目标

当前版本暂不实现：

- 自动定时同步知识库。
- 多网站聚合搜索。
- 用户画像记忆。
- 后台数据看板。
- 多轮复杂任务执行。

---

## 5. 用户故事

### 用户故事 1：站内知识问答

作为网站访客，我想直接询问“RAG 是什么”，系统可以根据站内文章解释概念，并给出来源。

### 用户故事 2：知识库未收录内容补充

作为网站访客，我询问一篇新文章中的内容，即使该文章没有提前进入知识库，系统也可以通过 sitemap 找到并读取该文章。

### 用户故事 3：无答案拒答

作为用户，我询问网站没有的内容时，系统不应该编造答案，而应该明确说明没有找到。

---

## 6. 核心流程

```text
用户提问
↓
知识库检索
↓
LLM 判断知识库结果是否足够
↓
如果足够：基于知识库回答
↓
如果不足：读取 sitemap
↓
从 sitemap 选择相关文章 URL
↓
Jina Reader 读取文章详情页
↓
判断文章内容是否足够
↓
如果足够：基于实时文章回答
↓
如果不足：无答案拒答
```

---

## 7. 功能需求

### 7.1 知识库问答

- 系统应支持接入 Dify Knowledge。
- 系统应根据用户问题检索知识库。
- 系统应判断检索结果是否足够回答。

### 7.2 sitemap 实时补充

- 当知识库不足时，系统应读取网站 sitemap。
- 系统应从 sitemap 中选择最相关的文章详情页 URL。
- 系统应过滤首页、标签页、分类页、归档页、分页页等无效 URL。

### 7.3 Jina Reader 网页读取

- 系统应使用 Jina Reader 读取文章详情页。
- 系统应将网页内容转换为可供 LLM 使用的 Markdown / 文本。

### 7.4 回答生成

- 系统只能基于知识库或实时读取的站内页面回答。
- 系统应在回答末尾展示参考来源。
- 系统不得基于站外知识自由发挥。

### 7.5 无答案拒答

- 如果知识库和站内页面均无相关内容，系统应明确拒答。
- 推荐拒答文案：

```text
我在当前知识库和站内页面中没有找到明确答案。
```

---

## 8. 非功能需求

| 类型 | 要求 |
|---|---|
| 准确性 | 回答必须基于知识库或站内页面 |
| 可追溯性 | 回答应提供来源 |
| 稳定性 | 判断节点输出应尽量稳定 |
| 安全性 | 不提供规避平台规则、绕过风控等建议 |
| 可维护性 | Prompt、测试集、badcase 需文档化 |
| 响应速度 | 普通知识库命中应优先直接回答，减少实时抓取次数 |

---

## 9. 评估指标

| 指标 | 定义 |
|---|---|
| 知识库命中率 | 问题可由知识库直接回答的比例 |
| 实时补充成功率 | 知识库不足时，sitemap + Jina 成功回答的比例 |
| 无答案拒答准确率 | 网站无内容时正确拒答的比例 |
| 幻觉率 | 回答中出现无依据内容的比例 |
| 来源引用覆盖率 | 回答提供参考来源的比例 |
| badcase 修复率 | 已发现问题中完成修复的比例 |

---

## 10. 风险与边界

| 风险 | 说明 | 应对方式 |
|---|---|---|
| 相关性误判 | 泛相关内容被判为可回答 | 优化判断 Prompt |
| 操作类误拒答 | 有操作步骤但被判为不足 | 增加操作类判断规则 |
| 平台合规风险 | 给出规避平台规则的建议 | 加入合规边界 Prompt |
| URL 选择错误 | sitemap 中选到标签页或分类页 | 过滤无效路径 |
| 来源展示差 | URL 过长或带符号 | 增加 URL 清洗 |
