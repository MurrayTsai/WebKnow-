# 测试用例与评估记录

## 1. 测试目标

本测试集用于验证 WebKnow 的三条核心路径：

1. 知识库命中 → 直接回答。
2. 知识库不足 → sitemap + Jina 实时补充。
3. 知识库和站内页面均无内容 → 明确拒答。

同时评估：

- 检索相关性判断
- 无答案拒答
- 跨概念对比
- 操作类问答
- 合规风险
- 来源引用质量

## 3. 详细记录

### Case 1：什么是 RAG？

- 类型：知识库不足但站内文章存在
- 实际路径：知识库判断 NO → 读取 sitemap → 选择 RAG 相关文章 → 读取文章详情页 → 实时回答
- 结果：通过
- 价值：验证 sitemap + Jina 实时补充分支可用。

### Case 2：什么是电商管理？

- 类型：知识库命中
- 实际路径：知识库检索 → 判断 YES → 知识库回答
- 结果：通过
- 观察：知识库没有严格定义，但系统能基于操作资料进行归纳。

### Case 3：VPN 和工具怎么用？

- 类型：操作类问题
- 实际路径：知识库判断 NO → sitemap 补充 → 无答案回复
- 结果：需优化
- 问题：知识库中存在相关操作内容，但判断节点过于严格。
- 优化：在判断 Prompt 中加入“操作步骤、工具名称、注意事项存在即可判断 YES”。

### Case 4：html5 超链接是什么？

- 类型：站内文章实时补充
- 实际路径：知识库判断 NO → sitemap 选择 HTML5 文章 → 读取正文 → 回答
- 结果：通过
- 价值：验证技术文章类问题可以通过 sitemap 补充回答。

### Case 5：TikTok 违规提示怎么办？

- 类型：合规风险测试
- 实际路径：知识库命中 → 直接回答
- 结果：流程通过，回答需优化
- 问题：回答中出现重新注册账号、VPN 等可能规避平台规则的建议。
- 优化：加入平台违规、封禁、风控类问题的合规回答边界。

### Case 6：AI 编造事实怎么办？

- 类型：相关性误判
- 实际路径：知识库判断 YES → 知识库回答
- 结果：需优化
- 问题：检索结果只是泛 AI 内容，与“编造事实/幻觉”无关。
- 优化：判断 Prompt 增加“同大类但未命中核心问题应输出 NO”。

### Case 7：RAG、PAL 和 ReAct 的区别是什么？

- 类型：跨概念对比
- 实际路径：知识库判断 NO → sitemap → 读取相关文章 → 回答
- 结果：通过
- 价值：验证系统可处理复杂对比型问题。

### Case 8：Prompt 通常包含几个关键部分？

- 类型：站内 AI 笔记类问题
- 实际路径：知识库判断 NO → sitemap → 读取 Prompt 文章 → 回答
- 结果：通过
- 观察：来源 URL 需要进一步格式清洗。

### Case 9：注册 Outlook 邮箱时国家地区选择哪个国家地区？

- 类型：知识库命中
- 实际路径：知识库检索 → 判断 YES → 知识库回答
- 结果：通过
- 观察：涉及账号、地区、平台环境时应增加合规边界。

### Case 10：重定义锅应该煮什么？

- 类型：无答案拒答
- 实际路径：知识库判断 NO → sitemap 未找到 URL → 拒答
- 结果：通过
- 价值：验证系统不会编造网站不存在的信息。

---

## 4. 测试结论

10 条测试中：

- 7 条通过。
- 3 条暴露出优化点。
- 三条核心路径均已验证：知识库回答、实时补充、无答案拒答。

主要优化方向：

1. 强化知识库相关性判断。
2. 降低操作类问题误拒答。
3. 增加平台合规边界。
4. 优化来源 URL 展示。
